# 用户旅程 — v1

## 主路径（Happy Path）

```
首次启动
  → 3 步引导（可跳过）
  → 授权相册访问
  → 首页：选来源「相机胶卷」+ 目标「旅行」
  → 开始整理
  → 滑动：右滑归入 / 左滑标记删除，可撤销
  → 47 张全部处理完
  → 整理完成摘要页
  → 查看待删除 (9)
  → 确认删除 9 张
  → 返回首页
```

---

## 非正常路径

### 权限相关

| 场景 | 行为 |
|------|------|
| 用户拒绝相册权限 | 展示说明页 +「去设置开启」，无法进入整理 |
| 用户授予有限访问 | 仅展示已授权照片；顶部提示条 +「添加更多照片」 |
| 整理中权限被收回 | 暂停滑动，弹窗引导重新授权 |

### 数据相关

| 场景 | 行为 |
|------|------|
| 来源相册待整理 0 张 | 禁用「开始整理」，显示「已全部整理」+「重新整理」 |
| 来源相册本身为空 | 显示空状态：「此相册没有照片」 |
| 新建相册名已存在 | 提示「相册名已存在，请换一个」 |

### 操作相关

| 场景 | 行为 |
|------|------|
| 右滑加入相册失败 | 卡片回弹，Snackbar 说明 + 重试，不跳下一张 |
| 左滑写库失败 | 同上 |
| 批量删除部分失败 | 展示「成功 N / 失败 M」，失败项保留在队列 |
| 用户撤销上一张 | 卡片恢复，DB 回滚，待删除队列同步更新 |

### 会话相关

| 场景 | 行为 |
|------|------|
| 整理到一半退出 App | 已处理记录已落库；再打开提示「继续整理？还剩 N 张」 |
| 用户选「重新整理」 | 清空该来源处理记录，全部照片重新进入队列 |

## 界面状态清单

### 滑动卡片（4 态）

| 状态 | 表现 |
|------|------|
| 加载中 | 占位骨架 / 加载指示器 |
| 就绪 | 全屏照片 + 浮层 |
| 滑动中 | 卡片跟随手势，左右提示色 |
| 失败 | 卡片回弹原位 + 错误提示 |

### 待删除页

| 状态 | 表现 |
|------|------|
| 空 | 「没有待删除的照片」+ 返回 |
| 有内容 | 网格 + 全选/确认删除 |
| 删除中 | 按钮 loading，禁止重复点击 |
| 部分失败 | 结果摘要 + 失败项高亮 |
