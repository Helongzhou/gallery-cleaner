# PRD — 相册整理 App v1

## 背景与目标

用户手机相册照片堆积，缺乏动力逐张整理。v1 用 Tinder/探探式滑动降低整理成本，让用户在碎片时间快速归类或标记删除。

**成功标准：**
- 用户能在 3 步内开始整理（选来源 → 选目标 → 开始）
- 单手滑动流畅，无明显卡顿
- 误删可挽回（撤销 + 待删除队列 + 系统最近删除）

---

## 功能清单

### F1 首页 — 相册选择

- 选择**来源相册**（系统相册列表 + 待整理数量）
- 选择**目标相册**（系统相册列表 + 新建相册）
- 「开始整理」按钮（待整理为 0 时禁用，显示「已全部整理」+「重新整理」）
- 有待删除照片时，底部显示「待删除 N 张」入口
- 有未完成会话时，提示「继续整理？」

### F2 滑动整理页

- 全屏照片 + 极简浮层（来源名、进度、操作提示、拍摄日期）
- 左滑 → 标记待删除，跳下一张
- 右滑 → 加入目标相册，跳下一张
- 滑动后 Snackbar 提示 + 撤销（3–5 秒，仅最近 1 次）
- 操作失败：卡片回弹，提示原因，可重试
- 屏幕尺寸缩略图 + 预加载下 2 张

### F3 整理完成摘要页

- 本次处理总数、归入数、待删除数
- 「查看待删除」「返回首页」「整理其他相册」

### F4 待删除确认页

- 缩略图网格，默认全选
- 取消勾选 / 恢复选中项
- 「确认删除」→ 二次确认弹窗 → 系统删除（进最近删除）
- 部分失败时展示成功/失败明细

### F5 首次引导

- 3 步引导页，仅首次显示，可跳过

### F6 权限与异常

- 相册权限请求（完全 + 有限访问）
- 有限访问时顶部提示 +「添加更多照片」
- 拒绝授权时引导去系统设置

### F7 持久化

- 每次滑动立即写入本地 DB
- 断点续整理
- 「重新整理」清空指定来源的处理记录

---

## 验收标准

1. 能从系统相册读取来源列表及照片数量
2. 能新建系统相册作为目标
3. 右滑后照片出现在目标系统相册中
4. 左滑后照片进入待删除队列，确认前不删除
5. 确认删除后照片进入 iOS「最近删除」
6. 已处理照片再次进入同来源时默认不出现
7. 撤销能恢复最近 1 次操作
8. App 中途退出再打开能续整理
9. 有限相册权限下 App 仍可用
